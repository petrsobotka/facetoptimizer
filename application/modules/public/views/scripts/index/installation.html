Installation

Requirements

FacetOptimizer is a PHP application build on top of Zend Framework and Doctrine 2, you will need PHP version 5.3 to run it.

Database backends

Because of Doctrine 2, FacetOptimizer can run backed by the following DB engines:

  • MySQL
  • MS SQL Server
  • Oracle
  • SQLite
  • PostgreSQL

There is a little SQL script model.sql to create appropriate database structure, but currently it is available only in MySQL dialect. If you wish to use another DB engine, please use attached model.clay, which is made using DB modeling tool called Azzurri Clay, and generate suitable SQL script for your favorite engine.

Installation

  1. Download the source code.
  2. Place it on your web server and make the /public directory a document root.
  3. Use /db/model.sql to create database structure.
  4. Create a configuration file inspired by /application/config/sample.php and then edit /application/config/config.php to match your config file by your server admin variable.
  5. If you plan to run FacetOptimizer in production:
    • It is highly recommended to use APC. After you install and configure APC, don't forget to set the corresponding variable in your config file to let FacetOptimizer know you want to use APC.
    • It is also highly recommended to run it on SSL secured host, especially if your traffic is considered sensitive.

Integration

After successful installation you will have to create a REST client inside your web application with faceted navigation. This client will connect to FacetOptimizer's API and capture traffic.

Please use any library for REST style communication you like. If your platform is PHP, we recommend Zend_Rest_Client.

Running unit tests

Unit tests are written using PHPUnit framework. Because of ZendFramework 1.11, the tests must be run under PHPUnit of version at maximum 3.5. The tests are stored in /tests folder, please run them using /tests/phpunit.xml configuraton file.
